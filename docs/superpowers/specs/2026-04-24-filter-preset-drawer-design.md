# 필터/검색 개선 설계: 프리셋 중심 오버레이 드로어

## 배경

현재 HBR 스타일 체커는 상단 검색창, 펼쳐진 `FilterPanel`, 스타일 카드 그리드로 구성되어 있다. 스타일 수가 많아질수록 필터 패널이 세로 공간을 차지하고, 사용자가 가장 많이 찾는 메타 조합/프리셋 진입점이 다른 필터와 같은 비중으로 보인다.

이번 개선은 프리셋/조합 탐색을 주 흐름으로 만들고, 원소/부대/티어 같은 빠른 필터는 보조 조작으로 유지한다. 카드 그리드 공간을 우선하기 위해 필터 패널은 기본적으로 숨기고 필요할 때만 오버레이 드로어로 연다.

## 사용자 경험

기본 화면은 카드 탐색을 우선한다. 헤더 아래에는 검색창, `필터` 버튼, 현재 적용 조건 요약 바만 표시한다. 요약 바에는 선택된 프리셋/메타 조합, 주요 필터, 숨김/흐림 모드, 결과 개수를 한 줄로 보여준다.

사용자가 `필터` 버튼을 누르면 좌측 오버레이 드로어가 열린다. 드로어는 카드 그리드 위에 떠서 표시되며, 뒤쪽 카드 영역은 약하게 어둡게 처리한다. 드로어 최상단에는 프리셋/조합 목록을 배치하고, 그 아래에 원소, 부대, 티어, 표시 모드 같은 빠른 필터를 둔다.

모바일에서는 좌측 드로어가 좁게 고정되지 않고 화면 폭 대부분을 사용하는 패널로 열린다. 닫기는 닫기 버튼, 바깥 영역 클릭, `Esc` 키로 가능해야 한다.

## 구조와 상태

기존 상태 구조는 최대한 유지한다.

- `App.jsx`의 `filters`, `searchTerm`, `activeMetaTeam`, `viewMode`는 그대로 사용한다.
- 새 상태로 `isFilterDrawerOpen`을 추가한다.
- `FilterPanel.jsx`는 항상 보이는 상단 패널이 아니라 드로어 내부 콘텐츠 역할로 바꾼다.
- 새 요약 UI는 현재 `filters`, `activeMetaTeam`, `viewMode`, 검색 결과 수를 받아 표시한다.
- 필터링 로직과 `StyleCard`의 `isDimmed`, `isHidden`, `isMetaHighlight`, `ownedCount` 동작은 변경하지 않는다.

## 컴포넌트 설계

`App.jsx`는 전체 레이아웃과 드로어 열림 상태를 관리한다. 검색창 아래 또는 같은 행에 필터 버튼과 요약 바를 렌더링하고, 필터 버튼 클릭 시 드로어를 연다.

`FilterPanel.jsx`는 프리셋/조합 섹션을 첫 번째 섹션으로 표시한다. 기존 원소, 부대, 티어, 숨김 모드 토글은 유지하되 드로어 안에서 세로로 읽기 쉽게 정리한다. 닫기 버튼은 드로어 헤더에 둔다.

필터 요약 UI는 별도 컴포넌트로 분리할 수 있다. 최소 표시 항목은 활성 프리셋 이름, 선택된 원소/부대/티어 개수 또는 이름, 표시 모드, 현재 표시 결과 수다. 선택된 조건이 없으면 "전체 스타일"처럼 명확한 기본 문구를 보여준다.

## 동작 규칙

프리셋과 빠른 필터는 함께 적용된다. 프리셋을 선택하면 기존 `activeMetaTeam` 기반 하이라이트 동작을 유지하고, 빠른 필터는 카드 목록을 좁히는 조건으로 계속 사용한다.

표시 모드는 기존처럼 `dim`과 `hide`를 지원한다. 스타일 개수가 많다는 요구를 반영해 사용자가 `hide` 모드를 켰을 때 카드 그리드가 실제로 줄어드는 현재 동작은 유지한다.

드로어가 열린 상태에서도 카드 클릭으로 보유/돌파 수가 바뀌면 안 된다. 오버레이가 카드 영역 위 클릭을 막아야 한다. 드로어 내부 버튼만 상호작용 가능해야 한다.

## 테스트 기준

- 필터 버튼을 누르면 드로어가 열리고, 닫기 버튼/바깥 영역/`Esc` 키로 닫힌다.
- 프리셋 선택, 원소/부대/티어 필터, 숨김/흐림 모드가 기존과 같은 결과를 만든다.
- 요약 바가 활성 프리셋과 선택 필터, 표시 모드, 결과 개수를 반영한다.
- 드로어가 열린 상태에서 배경 카드 클릭이 보유 상태를 변경하지 않는다.
- 모바일 폭에서 드로어가 화면 밖으로 넘치지 않고, 카드 그리드가 읽을 수 있는 폭을 유지한다.
